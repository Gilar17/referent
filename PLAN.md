# План: кнопки «О чем статья?», «Тезисы», «Пост для Telegram»

Опора: `PROJECT.md`. Сейчас парсинг и кнопка «Перевод» уже работают; для `summary` / `theses` / `telegram` API только возвращает сырой JSON статьи без вызова ИИ.

## Порядок действий

### 1. Зафиксировать контракт действий ✅
- Типы и форматы ответов зафиксированы в `lib/actions.ts`.
- `summary` — 1–3 абзаца сути на русском.
- `theses` — список ключевых тезисов на русском.
- `telegram` — готовый пост для копирования в Telegram.
- UI и API используют общий контракт (`isAction`, `ANALYZE_ACTION_CONTRACTS`).

### 2. Вынести общую подготовку текста статьи ✅
- Общая сборка текста вынесена в `lib/article-text.ts` (`buildArticleText`, `MAX_CONTENT_LENGTH`).
- `lib/translate-article.ts` использует общий хелпер; новые AI-действия (этап 3) подключают тот же вход.

### 3. Добавить AI-функции для трёх действий ✅
- Модуль `lib/analyze-article.ts`: `analyzeArticle` + `summarizeArticle` / `extractTheses` / `createTelegramPost`.
- Промпты по контракту из `lib/actions.ts`, вход через `buildArticleText`, вызов `chatCompletion`.
- Ответ на русском, без мета-комментариев, только по тексту статьи.

### 4. Подключить действия в API ✅
- В `app/api/analyze/route.ts` после парсинга:
  - `translate` → `translateArticle` → `{ result }`;
  - `summary` / `theses` / `telegram` → `analyzeArticle` → `{ result }`.
- Ошибки парсинга и OpenRouter по-прежнему как `{ error }` со статусом 422.

### 5. Обновить UI результата ✅
- Все действия показывают `data.result` как обычный текст.
- Статусы загрузки: перевод / краткое описание / тезисы / пост для Telegram.

### 6. Проверка
- Локально: валидный URL англоязычной статьи → каждая из трёх кнопок даёт осмысленный русский текст в блоке «Результат».
- Пустой/невалидный URL — прежние клиентские ошибки.
- Статья без контента / сбой OpenRouter — понятное сообщение об ошибке.
- Убедиться, что «Перевод» не сломан после рефакторинга общей сборки текста.

## Зависимости (уже есть)
- Парсинг: `lib/parse-article.ts`
- OpenRouter: `lib/openrouter.ts`, `OPENROUTER_API_KEY` в `.env.local`
- UI-кнопки и маршрут `/api/analyze` — готовы, нужна только AI-ветка и отображение `result`
